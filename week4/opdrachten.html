Project: MusicCollectionMVVM

Kom je er niet uit? Kijk nog eens goed naar de lesstof op het webdictaat!

CR-Update-D

De Create en Read actie van onze CRUD werken. Nu moet de Update actie nog geïmplementeerd worden. Zorg ervoor dat je op een element in de lijst kan klikken, en de waardes van dit element in de velden van Add new Song terecht komen.

Tip: Maak gebruik van de property SongViewModel op het SongListViewModel.

CRU-Delete

De Create, Read en Update actie van onze CRUD werken. Nu moet de Delete actie nog geïmplementeerd worden. Zorg ervoor dat de MusicCollection Applicatie deze functionaliteit ook ondersteunt. Voeg een knop toe aan de applicatie waarmee een geselecteerd liedje verwijderd wordt uit de lijst.

Daarnaast moet je de huidige selectie kunnen clearen zodat het weer mogelijk is om een nieuwe Song toe te voegen. Voeg een knop toe aan de applicatie waarmee de huidige selectie leeg wordt.

Tip: Maak gebruik van een RelayCommand

Voor de mensen die van een uitdaging houden!

Implementeer de onderstaande functionaliteit:

  • Disable de Add button als de velden leeg zijn of als er een bestaand liedje geselecteerd is.
  • Disable de Delete button als er geen bestaand liedje geselecteerd is.

Om de bovestaande functionaliteit te implementeren zal je de implementatie van de RelayCommand moeten aanpassen. Deze ondersteunt op dit moment nog geen werkende CanExecute methode. Implementeer deze functionaliteit via een een delegate.

Music Player

MvvM biedt ons de mogelijkheid om meerdere views op een view model aan te sluiten. Maak een tweede view voor de MusicCollection waarmee je door de liedjes heen kunt bladeren. Maak een nieuwe Window aan in het project en geef in de App.XAML aan dat je gebruik wil gaan maken van deze Window. Zorg ervoor dat je met een Vorige en Volgende knop door de collectie met liedjes kan heen bladeren.

Er zijn aanpassing op het View Model nodig om dit voor elkaar te krijgen.

Challange accepted!

Verzin een manier om te kunnen switchen tussen de twee views in de applicatie (MainWindow.Xaml en de Window uit de vorige opdracht). Allebei de windows moeten gebruik maken van hetzelfde View Model (als je in de ene view een nieuw liedje toevoegt, moet deze in de andere view in de lijst komen te staan).

Voor deze huiswerk opdracht mag je zelf een onderwerp verzinnen waar je een CRUD applicatie voor gaat bouwen. Voor de eerste opdracht moet je een CRUD applicatie maken met de volgende eisen:

My CRUD App

  • Maak gebruik van een extern MvvM framework (bijv. MvvM Light)
  • Gebruik een ViewModelLocator
  • De Applicatie heeft 1 window met de volgende functionaliteit:
    • Het tonen van een lijst van elementen
    • Het toevoegen van een nieuw element
    • Het wijzigen van een bestaand element
    • Het verwijderen van een bestaand element

De manier waarop je de applicatie vorm geeft mag je zelf invullen. Mogelijke onderwerpen voor de CRUD applicatie zijn:

  • Een kook boek met recepten
  • Een collectie van quotes van je vakdocent
  • Een TODO list

Tweede View

Voeg een tweede View toe aan je applicatie waarin je met een vorige en volgende knop door de collectie van je CRUD applicatie heen kan bladeren. Maak gebruik van het zelfde View Model en Model. Het moet ook mogelijk zijn om vanuit de nieuwe view gegevens van elementen te kunnen wijzigen.
